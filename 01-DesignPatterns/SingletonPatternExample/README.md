# Exercise 1: Implementing the Singleton Pattern

## Objective

Implement the Singleton Design Pattern for a Logger utility class and ensure that only one instance of the Logger is created throughout the application lifecycle.

## Files Included

* Logger.java
* TestLogger.java
* Output.png

## Description

The Logger class follows the Singleton Design Pattern by:

* Using a private constructor.
* Maintaining a private static instance.
* Providing a public static getInstance() method to access the single instance.

## Output

The output confirms that:

* The Logger object is created only once.
* Multiple references point to the same Logger instance.

## Result

Successfully implemented and tested the Singleton Pattern in Java.
